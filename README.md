# 拾光筛片

面向三下乡宣传组的本地图片筛选、最终选片、批量改名与 ZIP 导出工具。所有原图、缩略图、分类和压缩操作都在浏览器本地完成，不需要登录、后端或 AI API。

## 启动

要求 Node.js 20.19+（Vite 7 的最低要求）。

```bash
npm install
npm run dev
```

浏览器打开终端显示的本地地址。生产构建与测试：

```bash
npm run build
npm run test
npm run preview
```

## 直接部署到 GitHub Pages

本项目已经包含 `.github/workflows/deploy.yml`，不需要修改仓库名或 Vite 路径。

1. 在 GitHub 新建一个仓库，默认分支使用 `main`。
2. 将压缩包内的全部文件上传到仓库根目录，必须包含隐藏的 `.github` 文件夹。
3. 打开仓库的 **Settings → Pages**，在 **Build and deployment → Source** 中选择 **GitHub Actions**。
4. 打开 **Actions** 页面，等待“部署拾光筛片到 GitHub Pages”显示绿色勾号。
5. 回到 **Settings → Pages** 查看网站地址。以后每次推送到 `main` 都会自动重新测试、构建和部署。

如果上传文件时看不到 `.github` 文件夹，建议使用 Git 命令提交整个解压目录，不要逐个挑选可见文件。

## 使用流程

1. 在导入页填写项目名称，拖入 ZIP，或直接选择多张 JPG/JPEG/PNG/WebP 图片。
2. 在筛选页使用网格批量分类；点击照片进入单图模式，使用 `1`（废片）、`2`（待保留）、`3`（好片）快速判断。
3. 在最终选片页从好片和待保留中勾选照片，并在右侧拖动调整顺序。
4. 在改名与导出页逐张改名，或配置前缀、起始编号、位数和分隔符批量命名。
5. 导出最终选片 ZIP，或按“好片 / 待保留 / 废片 / 未分类”目录导出全部原图。

## 快捷键

- `1` / `2` / `3`：废片 / 待保留 / 好片
- `←` / `→`：上一张 / 下一张
- `Ctrl/Cmd + Z`：撤回
- `Ctrl/Cmd + Shift + Z`：重做
- `空格`：切换适应窗口与放大查看
- `Esc`：关闭大图

输入框、文本域或下拉框聚焦时不会触发快捷键。

## 主要目录

```text
src/
├── components/  # 网格、照片卡片、大图查看、排序、改名、导出等组件
├── pages/       # 导入、筛选、最终选片、改名与导出页面
├── stores/      # Zustand 状态、历史记录、撤回与重做
├── db/          # Dexie / IndexedDB 项目持久化
├── services/    # ZIP、缩略图、文件名、导出服务
├── hooks/       # Object URL 生命周期与快捷键
├── types/       # 核心类型
└── test/        # Vitest 单元测试
```

## 本地数据与隐私

- 原图以 `Blob` 存放在 IndexedDB，不转换成 Base64。
- 预览通过 `URL.createObjectURL` 创建，组件卸载或图片变化时会撤销 URL。
- 网格只渲染可视区域附近的缩略图；导出始终读取原始 Blob。
- 删除项目会同时删除 IndexedDB 中的原图、缩略图和编辑状态。
- 浏览器无痕模式、清理站点数据或存储配额不足会影响项目恢复能力。

## 性能边界

设计目标是 100–300 张普通手机或相机照片。ZIP 解压和缩略图生成采用异步逐项处理并主动让出主线程，网格按行虚拟化。当前版本没有把 ZIP 解压和 Canvas 缩略图处理移入 Web Worker，因此超大 RAW 转换图、数百张 40MP 以上照片或数 GB 压缩包仍可能受设备内存、IndexedDB 配额和浏览器单线程解码能力限制。Safari/iPadOS 的可用存储配额通常比桌面 Chromium 更紧张。

## 支持与限制

- 支持 ZIP 内嵌套目录、中文路径和同名图片；导入时保留原始完整路径作为内部信息。
- ZIP 的原目录结构不会保留，按文件状态统一导出。
- 不支持加密或密码保护 ZIP。
- 旋转、缩放和拖动仅改变预览，不改写原图像素。
- 导出期间请勿关闭页面；生成完成后会自动释放下载 Object URL。

## 自动验证覆盖

Vitest 覆盖扩展名识别、非法字符清理、扩展名保留、批量编号、大小写重名检测、分类与批量分类撤回、重做、批量改名撤回、最终顺序与移除后的顺序稳定性、ZIP 路径、原始 Blob 导出、10 张嵌套路径图片 ZIP 导入，以及 IndexedDB 项目恢复和删除。
